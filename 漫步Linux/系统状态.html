# 系统状态

# top

系统平均负载就是处于可运行( R : 使用 cpu 或者等待使用 cpu)或不可中断(D : 正在等待某些 io 访问,一般是和硬件交互,不可被打断,不可被打断是为了保护系统数据一致)的平均进程数.cpu 密集计算任务较多会导致平均负载上升,io 密集型任务较多会导致平均负载高但是 cpu 使用率不一定高,可能很低因为许多进程处于不可中断状态等待 cpu 调度也会升高负载.

  1. 第一行 : 当前时间、系统运行时间、当前多少用户、过去 1 分钟、5 分钟、15 分钟系统负载
  2. 第二行 : 当前有多少进程、正在运行中的进程、正在休眠的进程、停止的进程、僵尸进程, running 越多,服务器自然压力大
  3. 第三行 : 用户进程占 cpu 使用率、系统进程占 cpu 使用率、用户进程空间改变过系统优先级、空闲 cpu 占用率、等待输入输出的 cpu 时间比、硬件中断请求、软件的中断请求、steal time,us 长期过高表示用户进程占用率大量的 cpu 时间.
  4. 第四行 & 第五行 : 内存总量、空闲内存、已使用内存、写缓存、读缓存

# 列含义

top 采样依赖/proc/stat和/proc//stat

列 作用
VIRT 使用虚拟内存总量 VIRT=SWAP+RES
RES 进程使用的、违背换出的物理内存大小, RES=CODE+DATA
SHR 共享内存大小

# vmstat

vmstat -n n1 n2 # 第一个参数表示采样时间间隔,第二个参数表示采样次数
procs # r 运行和等待 cpu 的进程数一般运行队列不要超过总核数的 2 倍,否则压力过大 b 等待资源进程数,一般等待网络 io 或磁盘 io
cpu # us 用户进程消耗的 cpu 百分比,长期大于 50% 需要优化程序 sy 内核进程消耗的 cpu 百分比
1
2
3

# 结果

  1. procs
    • r : 运行和等待 cpu 时间片的进程数
    • b : 表示阻塞的进程数
  2. memory
    • swpd : 表示虚拟内存使用情况
    • free : 表示当前空闲的物理内存
    • buff : 表示缓存的内存大小
    • cache : 表示缓存的内存大小
  3. swap
    • si : 表示有磁盘读入内存大小
    • so : 表示内存写入磁盘大小
  4. io
    • bi : 表示由块设备读入数据的总量
    • bo : 表示写到块设备的总量
  5. system
    • in : 表示每秒中断数
    • cs : 表示每秒产生的上下文切换次数(线程切换)
  6. cpu
    • us : 表示拥护进程消耗的 cpu 时间百分比
    • sy : 表示系统调用消耗的 cpu 时间百分比
    • id : 表示 cpu 处于空闲状态的时间百分比
    • wa : 表示等待 io 的 cpu 时间

# 信号量

值 信号 说明
1 HUP 终端断线,终端被终止时会向所有关联进程发送该信号进程收到该信号后会终止运行,nohup 忽略该信号
2 INT 中断,ctrl + C
3 QUIT 退出,ctrl + \
9 SIGKILL 该信号不允许忽略和阻塞,可能会丢失数据或终端无法恢复正常状态
15 SIGTERM 默认 kill 信号该信号可以被执行、阻塞、和忽略,应用程序收到后可以做一些准备工作在进程程序终止
18 CONT 继续
19 STOP 暂停,ctrl + Z

kill 信号是直接发送给 init 进程的,由 init 进程负责终止 pid 指定的进程.
