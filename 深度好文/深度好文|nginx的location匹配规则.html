# nginx的location匹配规则

# 概述

官方给出的location语法如下

location [=|~|~*|^~] uri {...}
1

方括号中的四个匹配规则是可选项.uri是待匹配的请求字符串,可以包含正则也可以不包含正则.

# 四种可选标识符

标识符 描述
无 普通匹配,最长字符串匹配,与location顺序无关,是按照匹配的长短来取结果
= 精确匹配,要球请求字符串和uri严格匹配
~ 正则匹配,用于正则uri前,表示uri包含正则,且区分大小写
~* 正则匹配,用于正则uri前,表示uri包含正则,不区分大小写
^~ 非正则匹配,用于标准uri前,匹配到前缀最多的uri就结束,该模式成功后不会正则匹配
  1. 如果uri里包含正则,就必须使用~或~*标识符
  2. 针对~和~*可以在前面加上!来取反
    1. !~正则不匹配,区分大小写
    2. !~*正则不匹配, 不区分大小写

# 匹配规则

  1. 精确匹配,匹配成功后立即结束
  2. 正则匹配,区分大小写,匹配成功后立即结束
  3. 正则匹配,不区分大小写,匹配成功后立即结束
  4. 非正则匹配,匹配符合后,停止搜索正则
  5. 普通匹配与location顺序无关

匹配顺序 ( location = ) > ( location 完整路径 ) > ( location ^~路径 ) > ( location ~,~*正则顺序 ) > ( location 部分起始路径 ) > ( location / )

即 (精确匹配) > (最长字符串匹配,完全匹配) > (非正则匹配) > (正则匹配) > (最长字符串啤啤,不完全匹配) > (location通配符)
