# trace 实战

# 场景

trace 是基于事件的而不是基于采样的.

  • 并行执行度不足,比如没有充分利用多核资源
  • 因 GC 导致延迟较大
  • Goroutine 执行情况分析,发现各种阻塞(锁竞争、系统调用、调度、辅助 GC)导致的运行时短或延迟问题

# 添加 trace

# 手动添加

// runtime/trace
func main() {
    trace.Start(os.Stderr)
    defer trace.Stop()
    ch := make(chan string)
    go func() {
        ch <- "trace 实战"
    }()
    <-ch
}
// go tool trace trace.out

# goroutine 分析

能通过这个功能呢看到整个运行过程每个函数块有多少 goroutine,同时也可以看到 当前 goroutine 在整个调用好使中的占比,及 GC 清扫和 GC 暂停等待的开销.可以得知到底哪慢,然后决定下一步排查方向.

名称 含义 耗时
Goroutine goroutine ID 19
Execution Time 执行时间 17µs
Network wait 网络等待时间 0ns
Sync block 同步阻塞时间 0ns
Blocking syscall 调用阻塞时间 0ns
Scheduler wait 调度等待时间 15µs
GC sweeping GC 清扫 0ns (0.0%)
GC pause GC 暂停 0ns (0.0%)

# 查看跟踪

  1. 时间线 : 现实执行时间单元,根据时间维度不同可以调整区间
  2. 堆 : 显示执行期间的内存分配和释放情况
  3. 协程 : 显示执行期间每个 goroutine 运行阶段有多少个协程在运行包含 GC 等待(GCWaiting)、可运行(Runnable)、运行中(Running)三种状态
  4. os 线程: 显示在执行期间有多少个线程在运行,包含正在调用 syscall(InSyscall)、运行中(Running)两种状态
  5. 虚拟处理器 : 每个 p 显示一行,一般默认为系统内核数
  6. 协程和事件 : 显示在每个 p 上有什么 Goroutine 正在运行,连线行为代表事件关联.
名称 备注
Start 开始时间
Wall Duration 持续时间

End Stack Trace 结束时堆栈信息
Incoming flow 输入流
Outgoing flow 输出流
Preceding events 之前的事件
Following events 之后的事件
All connected 所有连接的事件
