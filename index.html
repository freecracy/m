# 态度

stay hungry ,stay foolish

随时饥饿,随时谦虚.

  1. 设计理念,为什么这样设计
  2. 解决问题,而且解决方法是最先进的
  3. 目标是什么,用什么方法呈现我的东西
  4. 把自己的工作写成论文,可以拿出去可以发表(价值),好到可以申请专利,好到可以发表论文
  5. 不要怕失败,失败是越早越好
  6. 先写测试再写代码

# 典型错误

  1. ppt 密密麻麻,答辩时念 ppt, -> ppt 言简意赅
  2. 讲得多,讲的深,拖时间, -> 留空白,留悬念,等评委问
  3. 答辩 : 关键点答,懂则懂,不懂就不懂,没有谁什么都懂,不编答案,直接回答不知道,不懂,不瞎扯
  4. 莫争执 : 评委都是对的,领导都是对的,答辩时即时终止话题
