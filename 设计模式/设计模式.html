# 设计模式

# 六原则

原则 用途
开闭原则 对修改关闭,堆扩展开放
单一职责 一个类只做一件事,一个类应该只有一个引起它修改的原因
里氏替换 子类应该可以完全替换父类,使用继承时只扩展新功能,而不破坏父类原有功能
依赖倒置 细节应该依赖抽象,抽象不应该依赖细节,抽象放在高处保持稳定,细节由底层实现层来完成
迪米特法则 又称最少知道原则,一个类不应该知道自己操作类的细节,只和朋友谈话,不和朋友的朋友谈话
接口隔离 客户端不应该依赖它不需要的接口,实现类只需只需依赖自己需要的接口方法

懒汉模式防止内存浪费,使用时在初始化

枚举实现

type Singleton struct{} // 单例结构体
var singleton *Singleton // 单例实体
var once sync.Once // 执行一次
// 返回单例
func GetInstance() *Singleton {
	once.Do(func() {
		singleton = &Singleton{}
	})
	return singleton
}

# 观察者模式

基于事件和响应.观察者模式中抽象出一个观察者接口,所有接受事件并做出响应的类都要实现该接口,事件发起者拥有一个观察者列表,可以注册新的观察者或删除观察者.

type subject struct {
	obsevers []observer
	context  string
}

func newSubject() *subject {
	return &subject{
		obsevers: make([]observer, 0),
	}
}

func (s *subject) attach(o observer) {
	s.obsevers = append(s.obsevers, o)
}

func (s *subject) notify() {
	for _, o := range s.obsevers {
		o.Update(s)
	}
}

func (s *subject) updateContext(context string) {
	s.context = context
	s.notify()
}

type observer interface {
	Update(*subject)
}

type reader struct {
	name string
}

func (r *reader) update(s *subject) {
	fmt.Printf("%s , %s \n", r.name, s.context)
}

func newReder(name string) *reader {
	return &reader{
		name: name,
	}
}
